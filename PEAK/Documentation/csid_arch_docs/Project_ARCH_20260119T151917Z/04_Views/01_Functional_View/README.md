# Functional View

**Purpose:** Describe what the system must do—independent of implementation.

## Include
- Key functions and decomposition
- Functional interfaces / exchanges
- Functional scenarios / chains
- Allocation hints (optional): functions to logical elements

## Prompts
- What are the top 5 mission functions?
- What functional sequences realize a primary operational scenario?
- Where are the major control points and information flows?

## “Good looks like”
- Function names are action-oriented and testable
- Scenarios connect to stakeholder needs
