# Architecture View Template

**Purpose:** Define how each view will be documented so the package stays consistent.

## Include
- Standard fields for a view:
  - Name / viewpoint
  - Stakeholder concerns addressed
  - Modeling notation/tools used
  - Key elements and relationships
  - Rationale and assumptions
  - Traceability links (requirements ↔ architecture)
- Definition of “done” for a view

## Prompts
- What does every view need so another team could reuse it?
- What is the “contract” between prose and model diagrams?

## “Good looks like”
- Repeatable and checklist-like
- Encourages traceability and review
