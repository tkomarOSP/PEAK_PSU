# Mapping Between Views

**Purpose:** Make traceability explicit across views (and to requirements).

## Include
- Function → Logical allocation table
- Logical → Physical allocation table
- Requirements → architectural element traces (if available)
- Gaps / TBDs explicitly listed

## Prompts
- What functions are not allocated yet?
- What logical components have no physical realization yet?
- What requirements have no architectural anchor?

## “Good looks like”
- Tables are complete enough to review
- TBDs are visible (not hidden)
