# System Overview

**Purpose:** Give readers enough context to understand the system before reading views.

## Include
- System mission / objectives
- Scope and boundary
- Stakeholders and their concerns
- Operational concept summary (high level)
- Architectural drivers (quality attributes, constraints)

## Suggested artifacts (optional)
- context diagram (image)
- stakeholder table
- glossary

## “Good looks like”
- A new reader can explain the system in 60 seconds
- Clear boundary: what’s in vs out
- Drivers are concrete (not vague aspirations)
