# Rationale

**Purpose:** Justify key decisions and tradeoffs; capture why the architecture looks this way.

## Include
- Architectural decisions (ADRs) or decision log
- Trade studies (even lightweight)
- Risks and mitigations
- Alternatives considered and why rejected
- Impact on quality attributes

## Prompts
- What decision, if wrong, would hurt us most?
- What constraints forced a design choice?
- Where do we need validation/experiments?

## “Good looks like”
- Decision rationale ties to drivers
- Alternatives are documented (prevents rework later)
