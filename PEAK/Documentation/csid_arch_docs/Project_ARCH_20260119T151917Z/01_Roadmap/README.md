# Roadmap

**Purpose:** Explain how this architecture package is organized and how the work will be developed over time.

## Include
- Project context (what problem are we solving?)
- Architecture development approach (phases, iterations)
- Deliverables and milestones
- Assumptions and constraints driving the plan

## Prompts
- What’s the minimum viable architecture description we can deliver in 2 weeks?
- What must be modeled vs documented in prose?
- What are the planned checkpoints for review (stakeholders, instructors, peers)?

## “Good looks like”
- Clear sequencing, not just a list
- Explicit assumptions and dependencies
- Review points mapped to stakeholders
